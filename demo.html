Core Concepts

1. Asset

Assets are single images uploaded by a user to be used to create new mash-up creations.

2. Library

Assets are sorted into libraries; users may create libraries that are public, private or belong to a group. Libraries may also belong to a group.

3. Creation

A creation is a combination of assets. Creations may be public, private or belong to a group. A user can also combine assets with an already existing creation to create a derivative.

4. Group

Users may create groups, and create invite links to allow other users to join their group. Groups may be public or private, and can have creations, libraries and consequently assets associated with them.

5. Moderation

All user-created content must be approved by a moderator before it is shown to any other users on the platform. Currently, this includes:

  • Assets
  • Libraries (title and description)
  • Creations
  • Groups
  • Users (usernames)
  • Comments

High-level architecture

flowchart

functions{Firebase\n functions}

auth{Firebase \n Auth}
user -->|uses| react
react -->|connects to| auth
functions -->|uses| firestore
functions -->|uses| auth
react -->|connects to| functions
react -->|uses| uploadcare

The React front-end uses the Firebase Authentication service to allow the user to login. All other actions the user takes are facilitated by the corresponding Firebase cloud functions, which interact with the Firestore database. Images are uploaded to and stored by Uploadcare, and the images' unique IDs are stored in the Firestore database.
